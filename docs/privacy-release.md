# AI 拍药盒识别隐私与发布检查

药盒、药瓶标签或处方照片可能包含姓名、医疗信息等敏感内容。AI 拍照识别功能上线前必须完成以下检查。

## 小程序内告知与同意

- 页面明确提示照片会上传至第三方 AI 服务处理，并提醒用户遮挡无关个人信息。
- 每次打开相机或相册前弹出处理说明；只有用户点击“同意继续”后才调用 `wx.chooseMedia`。
- AI 结果只作为草稿。用户必须对照药盒和医生处方确认药品、剂量与频次，未确认时禁止保存和创建提醒。
- 识别照片在发送第三方模型前会先由服务端解码并重新编码，去除 EXIF；默认不持久化。只有家属明确点击“保存主图”后，服务端才将处理后的照片绑定到当前家庭和选定药品。

## 微信公众平台配置

- 在小程序管理后台的《用户隐私保护指引》中声明“收集你选中的照片或视频信息”，并说明用途为药盒或处方信息识别与用药草稿录入。
- 确保隐私保护指引中的第三方处理说明与实际配置的模型供应商一致。
- 将 `https://api.0721online.net` 分别配置为 request、uploadFile 与 downloadFile 合法域名；发布前恢复域名校验并在真机完成识别上传和包装图下载验证。

微信公众平台配置不保存在代码仓库中，需要发布负责人逐项确认。

## 第三方模型服务

- 当前使用 GitHub Models。切换供应商时，应先核对服务条款、数据留存、训练使用、处理地域和删除机制。
- AI 识别流程只向第三方发送已重新编码且去除元数据的图片，不持久化识别上传；家属主动保存主图时，服务端才将处理后的图片存入 SQLite。不得在访问日志、错误日志或监控事件中记录图片正文、Base64 数据或处方内容。
- 包装主图使用短时签名地址读取。老人只应从本人提醒获得关联图片；家庭或药品删除后必须同步清理图片。
- `RECOGNITION_API_KEY` 只保存在服务器 `.env`，不得写入小程序、提交仓库或通过健康检查接口返回。

## 发布前验证

1. 用户拒绝照片处理说明时，相机和相册不会打开。
2. 同意后可以拍照或选择图片，上传域名配置有效。
3. AI 回填后未人工确认不能保存；修改药品、剂量或频次后需要重新确认。
4. 药库匹配失败时保留已经取得的识别草稿，不重复调用模型。
5. 伪造 MIME 的非图片会在调用模型前被拒绝；超过 5MB、上游超时、额度受限等错误不会保存用药记录。
6. 只有人工确认并选定药品后才能保存主图；老人端新提醒弹出对应照片，无照片时显示占位提示。
7. 验证主图上传/替换/删除权限、EXIF 清除、短时地址失效和家庭/药品删除级联清理。
